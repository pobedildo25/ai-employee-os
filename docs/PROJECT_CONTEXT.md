# Project Context — AI Employee OS

## Vision

Создать универсального AI-сотрудника для маркетингового агентства, который работает через Telegram и способен выполнять задачи как полноценный член команды.

Это **не чат-бот** с кнопками и сценариями. Это **агентная AI-система**, способная понимать намерения, планировать работу и использовать инструменты автономно.

## Целевая аудитория

- Маркетинговое агентство и его сотрудники
- Менеджеры проектов, стратеги, креативщики
- Клиенты агентства (косвенно — через результаты работы AI)

## Ключевые возможности

| Область | Что умеет AI Employee |
|---------|----------------------|
| Коммуникация | Принимает задачи через Telegram на естественном языке |
| Понимание | Определяет цель, сложность и необходимые действия без жёстких правил |
| Документы | Создаёт, анализирует, реверс-инжинирит структуру, генерирует по шаблонам |
| Презентации | Создаёт презентации с учётом фирменного стиля |
| Аудио | Транскрибирует и анализирует звонки через Whisper |
| Изображения | Анализирует визуальный контент |
| Память | Хранит контекст клиентов, проектов, предпочтений |
| Обучение | Учится на обратной связи и корректирует поведение |
| Бренд | Применяет фирменный стиль из брендбуков и исторических документов |

## Главный принцип

> Мы **не** строим систему из жёстких сценариев.

### Запрещено

```
если пользователь написал X → вызвать функцию Y
```

### Правильно

```
User Request
    ↓
Executive Agent          — понимает запрос, определяет цель
    ↓
Capability Planning      — какие способности нужны
    ↓
Task Planning            — план выполнения
    ↓
Skill Selection          — выбор конкретных skills
    ↓
Execution                — выполнение с очередью и retry
    ↓
Review                   — проверка качества
    ↓
Memory Update            — сохранение результата и контекста
```

## Пользовательские сценарии

### Простой запрос

> «Напиши краткое резюме встречи с клиентом X»

Executive Agent определяет: нужен контекст клиента + история проекта → Skill: Document Generator → результат без многошагового плана.

### Сложный запрос

> «Создай презентацию из звонка клиента»

1. Executive Agent: сложная задача, нужен план + approval
2. Capability Planner: Audio Processing, Whisper, Analysis, Presentation Generator
3. Planner: шаги — транскрипция → анализ → структура слайдов → генерация
4. Human Approval: пользователь подтверждает план
5. Task Executor: выполняет шаги
6. Reviewer Agent: проверяет качество
7. Memory Update: сохраняет артефакт и обновляет Client Memory

### Onboarding клиента

> Загрузка истории агентства: КП, презентации, отчёты, звонки, брендбуки

Система создаёт:
- **Client Profile** — структурированный профиль
- **Client Memory** — контекст для будущих задач
- **Knowledge Base** — семантический поиск по материалам
- **Semantic Rules** — правила и предпочтения клиента

## Ограничения и границы

- AI Employee **не заменяет** финальное решение человека в критичных задачах
- Сложные задачи проходят через **Human Approval Layer**
- Все артефакты **версионируются**
- Все действия **трассируются** (trace_id, логи, история шагов)

## Успех проекта

Система считается успешной, когда сотрудник агентства может:

1. Отправить задачу в Telegram обычным языком
2. Получить качественный результат без ручной настройки сценариев
3. Доверять памяти системы о клиентах и проектах
4. Видеть прозрачный процесс выполнения
5. Корректировать поведение через обратную связь
